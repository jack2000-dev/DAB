## Presentation rules

- **Self-question:** What's the single most important thing I want my audience to learn?
- Keep text < 5 lines and < 25 words per slide
- Most presentations: **30–60 minutes**
- No slang or unfamiliar abbreviations
- Pick one theme; stick to it
- Visual should speak for itself

## Best practices

- Attention-grabbing opening
- Broad → specific
- Short sentences
- Pause 5 seconds after showing a viz
- Pause intentionally
- Keep voice pitch level
- Stand still; move with purpose
- Good posture
- Look at audience (or camera)
- Concise message
- End by explaining why analysis matters

## Presentation flow

```
Purpose statement → Last update → Visual (with explanation) → Annotation → Text (one bullet at a time)
```

## Tips

1. Channel excitement
2. Start with the broader idea
3. "Let me know if anything is unclear."
4. Preparation is key
5. Share deck with agenda via email beforehand

## How to handle Q&A

1. Prepare expected questions (add to appendix)
2. Discuss deck with manager/peers
3. Ask what questions this audience usually asks
4. Seek pre-feedback on the deck
5. ≥24h before, brainstorm tricky questions
6. Practice presenting

## 30-minute agenda example

| Block | Time |
|-------|------|
| Introductions | 4 min |
| Project overview & goals | 5 min |
| Data and analysis | 10 min |
| Recommendation | 3 min |
| Actionable steps | 3 min |
| Questions | 5 min |

## McCandless Method

1. Introduce the graphic by name
2. Answer obvious questions before they're asked
3. State the insight
4. Call out data that supports the insight
5. Tell why it matters
